## Official Documentation

### PostgreSQL Documentation

**PostgreSQL Official Docs**
🔗 https://www.postgresql.org/docs/

The most comprehensive resource for PostgreSQL. Well-written, with examples.

**Key Sections**:
- **Tutorial**: Gentle introduction to SQL
- **SQL Language**: Complete SQL reference
- **Server Administration**: Configuration, performance
- **Internals**: How PostgreSQL works under the hood

**Start here**:
- Tutorial: https://www.postgresql.org/docs/current/tutorial.html
- SQL Syntax: https://www.postgresql.org/docs/current/sql-syntax.html

### pg_stat_statements

**What**: PostgreSQL extension for query statistics
**Use**: Find slow queries in production

```sql
-- Enable extension
CREATE EXTENSION pg_stat_statements;

-- View slow queries
SELECT
    query,
    calls,
    total_exec_time,
    mean_exec_time,
    max_exec_time
FROM pg_stat_statements
ORDER BY total_exec_time DESC
LIMIT 10;
```

### Recursive CTEs

**Resource**: PostgreSQL docs
🔗 https://www.postgresql.org/docs/current/queries-with.html

**Use cases**:
- Hierarchical data (org charts)
- Graph traversal
- Date series generation

**Example**: Generate Fibonacci sequence
```sql
WITH RECURSIVE fibonacci(n, fib_n, fib_n1) AS (
    VALUES (0, 0, 1)
    UNION ALL
    SELECT n + 1, fib_n1, fib_n + fib_n1
    FROM fibonacci
    WHERE n < 10
)
SELECT n, fib_n FROM fibonacci;
```

### Lateral Joins

**Resource**: PostgreSQL docs
🔗 https://www.postgresql.org/docs/current/queries-table-expressions.html#QUERIES-LATERAL

**What**: Subquery that can reference previous tables

**Example**: Top 3 orders per user
```sql
SELECT u.user_id, u.name, o.order_id, o.total
FROM users u
CROSS JOIN LATERAL (
    SELECT order_id, total
    FROM orders
    WHERE orders.user_id = u.user_id
    ORDER BY total DESC
    LIMIT 3
) o;
```

### JSON in PostgreSQL

**Resource**: PostgreSQL JSON docs
🔗 https://www.postgresql.org/docs/current/datatype-json.html

**Operators**:
- `->`: Get JSON object field
- `->>`: Get JSON object field as text
- `@>`: Contains
- `?`: Key exists

**Example**:
```sql
SELECT
    user_id,
    metadata->>'country' AS country,
    metadata->'preferences'->>'theme' AS theme
FROM users
WHERE metadata @> '{"country": "US"}';
```

### Full-Text Search

**Resource**: PostgreSQL text search docs
🔗 https://www.postgresql.org/docs/current/textsearch.html

**Example**:
```sql
-- Create tsvector
SELECT to_tsvector('english', 'The quick brown fox');

-- Search
SELECT * FROM articles
WHERE to_tsvector('english', content) @@ to_tsquery('english', 'fox & quick');

-- Create index
CREATE INDEX idx_articles_content ON articles
USING GIN (to_tsvector('english', content));
```

### Materialized Views for Analytics

**Pattern**: Pre-compute expensive aggregations

**Example**:
```sql
-- Create materialized view
CREATE MATERIALIZED VIEW daily_metrics AS
SELECT
    DATE(order_date) AS day,
    COUNT(*) AS order_count,
    SUM(total_amount) AS total_sales,
    AVG(total_amount) AS avg_order_value
FROM orders
GROUP BY DATE(order_date);

-- Create index
CREATE UNIQUE INDEX ON daily_metrics(day);

-- Refresh concurrently
REFRESH MATERIALIZED VIEW CONCURRENTLY daily_metrics;
```

## Recommended Learning Path

### 1. Absolute Beginner (Week 1-2)

- [ ] SQLBolt (complete all lessons)
- [ ] PostgreSQL tutorial (official docs)
- [ ] Practice with Chinook database

**Goal**: Write basic SELECT queries, understand JOINs.

### 2. Intermediate (Week 3-4)

- [ ] Mode SQL Tutorial (advanced sections)
- [ ] Read "SQL for Data Analysis" (first half)
- [ ] LeetCode Easy problems (20-30 problems)
- [ ] Practice aggregations and window functions

**Goal**: Complex queries with CTEs, window functions.

### 3. Advanced (Week 5-6)

- [ ] Read "SQL Performance Explained"
- [ ] Practice EXPLAIN on all queries
- [ ] Use The Index, Luke (complete guide)
- [ ] LeetCode Medium/Hard problems

**Goal**: Understand query optimization, indexing strategies.

### 4. Expert (Ongoing)

- [ ] Deep dive into PostgreSQL internals
- [ ] Read academic papers on query optimization
- [ ] Contribute to PostgreSQL community
- [ ] Build tools/extensions

**Goal**: Master-level understanding of SQL and databases.
